# ADR-001: 도메인 패키지 내부 구조

## 날짜
2026-05-11

## 상태
`결정됨`

---

## 상황

현재 각 도메인 패키지(member, product 등) 안에 API 컨트롤러와 View 컨트롤러가 구분 없이 혼재한다.

```
gift/member/
  MemberController.java       # REST API
  AdminMemberController.java  # Thymeleaf 뷰
  Member.java
  MemberRepository.java
  ...
```

파일이 늘어날수록 각 클래스의 역할을 이름만으로 파악해야 하며, 패키지만 봐서는 API인지 뷰인지 알 수 없다.

---

## 선택지

| 선택지 | 설명 | 장점 | 단점 |
|--------|------|------|------|
| A. 현행 유지 | 도메인 패키지 내 평탄 구조, 이름으로 구분 | 패키지 수 최소 | 역할이 패키지로 드러나지 않음 |
| B. 상위에 api/view 분리 | `gift/api/member/`, `gift/view/member/` | 레이어 구분 명확 | 도메인 응집도 깨짐, 한 도메인 수정 시 두 곳 이동 |
| C. 도메인 하위에 api/view 분리 | `gift/member/api/`, `gift/member/view/` | 도메인 응집도 유지 + 역할 구분 | 패키지 수 증가 |

---

## 결정

**C를 선택한다. 도메인 패키지 하위에 `api/`, `view/` 서브 패키지를 둔다.**

view가 없는 도메인(order, wish, category, option, auth)도 일관성을 위해 `api/` 하위 패키지를 적용한다.

```
gift/member/
  api/
    MemberController.java
    MemberRequest.java
    MemberResponse.java
  view/
    AdminMemberController.java
  Member.java
  MemberRepository.java

gift/order/
  api/
    OrderController.java
    OrderRequest.java
    OrderResponse.java
  Order.java
  OrderRepository.java
```

---

## 트레이드오프

감수하는 것: 패키지 depth 증가, 파일 탐색 시 한 단계 더 들어가야 함

얻는 것: 패키지 경로만으로 역할 파악 가능(`gift.member.api` vs `gift.member.view`), 나중에 view 제거나 API 분리 시 경계가 명확

---

## 결과

- 이 구조 변경은 작동 변경 없이 순수 리팩터링으로 진행

### api/로 이동 (Controller, Request, Response)

| 클래스 | 이동 후 패키지 |
|--------|---------------|
| `KakaoAuthController`, `TokenResponse` | `gift.auth.api` |
| `MemberController`, `MemberRequest` | `gift.member.api` |
| `CategoryController`, `CategoryRequest`, `CategoryResponse` | `gift.category.api` |
| `ProductController`, `ProductRequest`, `ProductResponse` | `gift.product.api` |
| `OptionController`, `OptionRequest`, `OptionResponse` | `gift.option.api` |
| `WishController`, `WishRequest`, `WishResponse` | `gift.wish.api` |
| `OrderController`, `OrderRequest`, `OrderResponse` | `gift.order.api` |

### view/로 이동 (Admin HTML 컨트롤러)

| 클래스 | 이동 후 패키지 |
|--------|---------------|
| `AdminMemberController` | `gift.member.view` |
| `AdminProductController` | `gift.product.view` |

### 도메인 루트에 유지

| 분류 | 클래스 | 이유 |
|------|--------|------|
| Entity | `Member`, `Product`, `Category`, `Option`, `Wish`, `Order` | 도메인 핵심 객체 |
| Repository | `*Repository` | 도메인 핵심 |
| 도메인 규칙 | `ProductNameValidator`, `OptionNameValidator` | 도메인 규칙 |
| 인증 인프라 | `JwtProvider`, `AuthenticationResolver`, `KakaoLoginClient`, `KakaoLoginProperties` | 인증 인프라 |
| 주문 인프라 | `KakaoMessageClient` | 주문 인프라 |
